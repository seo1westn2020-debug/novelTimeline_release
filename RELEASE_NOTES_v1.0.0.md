# novel-timeline v1.0.0 — 첫 공개 릴리즈

**릴리즈 날짜**: 2026-04-26

소설 작가를 위한 **타임라인 기반 스토리 관리 도구**의 첫 공개 빌드입니다. 등장인물·사건·복선·관계도·집필 에디터·자료 보관·AI 보조까지, 장편 원고 작업에 필요한 도구를 한 화면 안에서 사용할 수 있습니다.

---

## 새로 추가된 기능

### 인물 참고 사진
- 인물마다 **수집자료 사진을 최대 5장까지** 묶어둘 수 있습니다.
- 수집자료 탭에 사진을 한 번만 올려두면, 인물 편집창에서 골라 붙이는 방식이라 같은 파일을 두 번 업로드할 필요가 없습니다.
- 집필 에디터 우측 사이드바에서 인물 카드의 썸네일을 클릭하면 **원본 크기(화면의 90%)** 로 팝업이 떠서, 외모·의상·소품을 빠르게 확인하면서 원고를 쓸 수 있습니다.

### 인물 노드 보색 outline
- 다크 모드에서 인물 색이 배경과 겹쳐 잘 안 보이던 문제를 해결했습니다.
- 모든 인물 색상에 **자동으로 보색 테두리**가 들어가, 어떤 색을 골라도 관계도·인물 카드에서 또렷하게 보입니다.

### 햄버거 메뉴 도움말 (FAQ) 11개 항목으로 확장
- 우측 상단 햄버거 메뉴(≡) → "도움말"에서 자주 묻는 질문을 바로 확인할 수 있습니다.
- 체험 정책(3개월 + 읽기 전용 모드), 라이선스 키 입력 방법, 인물 사진 등록, 별칭(aliases) 활용, NTZ vs JSON 차이, 후원 채널, 버그 신고 채널, AI 기능 사용법(BYOK 비용 안내 포함) 등 11가지 주제를 안내합니다.

## 개선된 부분

### 첫 실행 화면이 깨끗한 빈 상태로 시작합니다
- 이전에는 첫 실행 시 자동으로 샘플 프로젝트가 떴지만, "샘플은 필요할 때 직접 만들면 된다"는 사용자 의견을 반영해 **빈 프로젝트**로 시작하도록 변경했습니다.
- 이미 작업해두신 프로젝트가 있다면 종전과 동일하게 그대로 불러와집니다.

### "+자료 추가" 버튼이 현재 필터에 맞춰 동작
- 수집자료에서 **사진 필터**를 선택한 상태로 "+자료 추가"를 누르면 디폴트 타입이 **사진**으로 자동 설정됩니다 (이전에는 항상 메모로 시작).
- "사진 추가하려고 눌렀는데 메모 폼이 떠서 헷갈리던" 함정이 사라집니다.

### Windows 빌드를 EXE 한 종류로 통일
- 이전 베타에서는 `.msi`와 `.exe` 두 가지 인스톨러를 제공했지만, "MSI vs EXE 차이를 일반 사용자가 알기 어렵다"는 판단으로 **NSIS .exe 한 종류로 통일**했습니다.
- 다운로드 페이지가 단순해졌고, 설치 안내도 한 가지로 모일 수 있게 되었습니다.

### 후원자 명단 GitHub 단일 출처화
- 인앱 후원자 모달을 GitHub 페이지의 "도움을 주신 분들 / Supporters" 섹션 링크로 대체했습니다.
- 후원·번역 교정·기능 제안·버그 제보·아이디어 공유까지 비금전 기여도 함께 게재되며, 한 곳에서만 관리되어 항상 최신 상태입니다.

## 수정된 버그

### 본문에만 언급된 인물도 충돌 검증에 포함됩니다 (BUG-012)
- **이전 동작**: 인물 X를 사건의 "등장인물" 필드에 태그하지 않고 **원고 본문에만** 이름을 적은 경우, AI 충돌 검증이 그 인물을 보지 못해 금기어 위반·말투 모순·관계 모순을 못 잡아내는 **무성 실패(silent failure)** 가 있었습니다. UI에는 "충돌 없음"만 표시됐습니다.
- **이번 수정**: 본문에 등장하는 인물명·별칭(aliases)을 자동 인식하여 검증 대상에 포함합니다. 이제 인물을 일일이 태그하지 않아도, 본문에 이름만 쓰면 페르소나 정보가 AI 프롬프트에 함께 들어갑니다.

### NTZ 가져오기 직후 이미지가 사라지던 문제 (BUG-014, Windows .exe 한정)
- **이전 동작**: Windows 빌드(.exe)에서 NTZ 파일을 가져오면 수집자료 항목 이름은 보이지만 **이미지 파일만 로드되지 않던** 문제가 있었습니다.
- **이번 수정**: Tauri 백엔드의 자산 저장/로딩 명령을 다시 정렬해, Windows 빌드에서도 NTZ 가져오기 후 이미지가 정상적으로 표시됩니다.

### 빈 프로젝트 상태에서 NTZ 가져오기가 막히던 문제 (BUG-015)
- **이전 동작**: 첫 실행 직후 **빈 프로젝트 상태**에서 NTZ 가져오기를 누르면 "저장 실패 — 프로젝트 제목이 비어있습니다" 팝업이 떠서 import가 막히던 회귀 버그.
- **이번 수정**: 빈 상태에서는 사전 자동 저장을 건너뛰고 바로 import가 진행되도록 보강했습니다.

### 라이선스 갱신 안전망 (BUG-011)
- 결제 직후 인터넷이 끊긴 환경(여행·외출 등)으로 들어가는 시나리오에서, 자동 갱신 웹훅 일부가 누락되더라도 **읽기 전용 모드로 잘못 전환되지 않도록** 만료 시각에 **3일 버퍼**를 추가했습니다.
- 정상 갱신 신호가 도착하면 그 값으로 덮어씌워지므로, 실제 만료 시각이 부풀려지지는 않습니다.

## 알려진 제약사항

- **코드 서명 인증서가 아직 적용되어 있지 않습니다.** Windows SmartScreen, macOS Gatekeeper의 우회 방법은 [README — 다운로드](./README.md#다운로드) 를 참고해 주세요.
- **AI 기능은 BYOK(Bring Your Own Key) 방식**입니다. OpenAI / Anthropic / Google 중 한 곳의 API 키를 설정 → AI에 입력해야 사용할 수 있습니다. 1회 검증 비용은 대략 **$0.005~0.01** 수준입니다.
- **체험판은 설치 후 90일** 입니다. 만료 후에는 **읽기 전용 모드**로 전환되며, 데이터 자체는 어떠한 경우에도 손상되지 않고 내보내기로 외부 백업이 가능합니다.

## 설치 방법

### Windows 10 / 11

1. [Latest Release](https://github.com/seo1westn2020/novelTimeline_release/releases/latest) 페이지에서 `novel-timeline_<버전>_x64-setup.exe` 파일을 다운로드합니다.
2. 다운로드한 파일을 더블클릭합니다.
3. SmartScreen 경고가 뜨면 작은 글씨의 **"추가 정보"** → **"실행"** 을 차례로 클릭합니다.
4. 안내에 따라 설치를 마치면 시작 메뉴와 바탕화면에 아이콘이 만들어집니다.

### macOS 12 이상

1. [Latest Release](https://github.com/seo1westn2020/novelTimeline_release/releases/latest) 페이지에서 본인 Mac에 맞는 dmg 파일을 받습니다.
   - **Apple Silicon** (M1/M2/M3/M4 등): `..._aarch64.dmg`
   - **Intel Mac**: `..._x64.dmg`
2. 받은 .dmg 파일을 더블클릭하면 마운트되며, novel-timeline 아이콘을 **응용 프로그램** 폴더로 끌어다 놓습니다.
3. 첫 실행 시 응용 프로그램 폴더에서 **아이콘을 우클릭 → "열기"** 를 선택합니다 (그냥 더블클릭하면 막힐 수 있습니다).
4. 보안 다이얼로그에서 한 번 더 **"열기"** 를 클릭하면 정상 실행됩니다.

> macOS Sequoia(15) 이상에서 위 방법으로도 막힐 경우, **시스템 설정 → 개인정보 보호 및 보안** 화면 하단의 **"확인 없이 열기"** 버튼을 사용해 주세요.
